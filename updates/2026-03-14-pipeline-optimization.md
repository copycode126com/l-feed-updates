我们对内容生成流水线和前端阅读体验进行了全面升级，打造更稳定、更高效的内容生态系统。

### 主要功能

**自动化内容调度系统**
- 每日三批次定时生成：早、中、晚各批次产出 18 篇文章，每天共 54 篇
- 基于 Go 的智能调度服务，5 秒轮询间隔确保任务精准执行
- ChromeDriver 进程管理优化，修复并发场景下的资源泄漏问题
- 指数退避重试机制：失败任务自动重试，延迟递增（1分钟、2分钟、4分钟...）

**多级内容导航系统**
- 新增一级分类（News、Explore）和二级子分类下拉筛选
- 导航状态自动保存，无缝衔接用户阅读体验
- 侧边栏三级菜单展开/收起功能，状态持久化存储
- 超过 1200 行 CSS 优化，界面更加精致流畅

### 其他优化

- **精确进程管理**：每个任务独立追踪 ChromeDriver PID，解决进程互相干扰问题
- **优雅关闭机制**：三步清理流程（SIGTERM → 等待 → SIGKILL）确保资源正确释放
- **事务化任务状态**：数据库事务机制防止并发执行时的竞态条件
- **完善日志记录**：记录所有任务执行时长、状态和错误信息，便于监控排查
- **技能化执行流程**：article-task-executor 技能自动化从任务创建到文章处理的完整工作流
